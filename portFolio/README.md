## 📌 Project Description

This project simulates a variety store that sells clothing and electronics using MongoDB.

The goal is to design a flexible NoSQL schema that allows storing different product types in the same collection.

Clothing products include:
- size
- color
- fabric

Electronics include:
- voltage
- warranty
- cpu

---

## 🧠 Why MongoDB?

MongoDB is ideal because:
- It supports polymorphic schemas
- It allows nested objects
- It provides flexible queries
- It eliminates rigid SQL tables

We use a single collection called:

products

With a dynamic field:

attributes
